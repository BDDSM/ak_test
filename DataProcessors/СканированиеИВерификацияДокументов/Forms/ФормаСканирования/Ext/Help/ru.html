Обработка "FILLIN:Загрузка документов"

Программное решение FILLIN:Загрузка документов может распознавать отсканированные документы и автоматически вносить полученные данные в базу 1С.

Основная форма обработки при запуске обработки из конфигурации Бухгалтерия предприятия 2.0. На форме находятся 3 основных страницы: "Значения по умолчанию", "Сканирование", "История".

  • Значения по умолчанию - на данной вкладке устанавливаются значения по умолчанию, которые будут подставлены в документ на стадии верификации в случае, если соответствующие поля не будут распознаны.
  • Сканирование - на данной вкладке представлены все отсканированные документы.
  • История - на вкладке отображены все документы, занесенные при помощи данной обработки, отсортированные по дате. При двойном клике по документу открывается соответствующий документ в базе данных 1С.

Значения по умолчанию:

На данной вкладке устанавливаются значения по умолчанию.

Вкладка, с которой будут подставлены значения (Покупка, Продажа) зависит от поля Вид документа.

Алгоритм определения Вида документа:

  • Если распознанное значение ИНН/КПП Заказчика из документа полностью совпадает с ИНН/КПП в справочнике Организации - вид документа Поступление.
  • Если распознанное значение ИНН/КПП Поставщика из документа полностью совпадает с ИНН/КПП в справочнике Организации - вид документа Реализация.
  • Если распознанное значение ИНН/КПП Поставщика из документа не найдено в справочнике Организации, но полностью совпадает с ИНН/КПП в справочнике Контрагенты - вид документа Реализация.
  • Если распознанное значение ИНН/КПП Заказчика из документа не найдено в справочнике Организации, но полностью совпадает с ИНН/КПП в справочнике Контрагенты - вид документа Поступление.

Если стоит галочка в поле Установливать в документе текущую дату, то при создании документа дата будет заменена на текущую.

Выбранное значение в поле Организация и Контрагент будет подставлено в документ, когда он будет экспортирован в 1С на вкладку Сканирование.

Установленное значение Организация/Контрагент не будет браться из Значений по умолчанию, если распознанное значение нашелось в базе в соответствующем справочнике.

Если для поля Организация нет нужного значения в справочнике, его можно добавить, нажав на кнопку Добавить организацию.

Если для поля Контрагент нет нужного значения в справочнике, его можно добавить, нажав на кнопку Добавить контрагента.

Сканирование:

Если сканер подключен локально, необходимо выполнить следующие действия:

  1. В окне "FILLIN: Загрузка документов" нажать кнопку Сканировать.
  2. Откроется приложение FILLIN COM SDK.
  3. Нажать Файл-->Настройки. Откроется окно настроек системы.
  4. Перейти на вкладку Импорт. На данной вкладке можно при необходимости установить флажки Автоматически отправлять когда документов с указанием количества документов (для автоматической отправки документов со стадии Импорт на последующие стадии) и Объединить в один документ при отправке (для объединения нескольких документов в один многостраничный документ).
  5. Выбрать тип цели Сканер, в поле Устройство указать необходимое устройство.
  6. Установить необходимое значение поля Разделение документов. Возможные значения поля Разделение документов:
    • Отключено - значение по умолчанию. Предполагает отключение функции разделения документов;
    • По штрих-коду - необходимо задать формат штрих-кода и значение;
    • По пустой странице - необходимо задать количество черных точек в процентном содержании;
    • По числу страниц с указанием количества страниц в документе.
  7. Нажать ОК.
  8. Отсканировать документы. Документы появятся в области отображения документов на стадии Импорт.
  9. Нажать кнопку Отправить. Документы будут отправлены на распознавание (стадия OCR). Документы обрабатываются с помощью описанных шаблонов, кроме того извлекаются значения для требуемых полей. После распознавания документа он автоматически перемещается На экспорт.
  10. Перейти на стадию Экспорт.
  11. Нажать кнопку Экспортировать. Окно системы FILLIN закрылось. Распознанные документы отобразились на данной форме на закладке Сканирование.
  12. Отметить галочкой поле Экспортировать, если документ необходимо верифицировать и записать в базу данных. Кнопка Не экспортировать ничего снимает галочки со всех документов на вкладке, кнопка Экспортировать всё ставит галочки у всех документов на данной вкладке.

Для того чтобы загрузить отсканированные документы из папки на локальном диске, необходимо выполнить следующие действия:

  1. В окне "FILLIN: Загрузка документов" нажать кнопку Сканировать.
  2. Откроется приложение FILLIN COM SDK.
  3. Открыть окно настроек системы.
  4. Перейти на вкладку Импорт. На данной вкладке при необходимости установить флажки Автоматически отправлять когда документов с указанием количества документов (для автоматической отправки документов со стадии Импорт на последующие стадии) и Объединить в один документ при отправке (для объединения нескольких документов в один многостраничный документ).
  5. Выбрать тип цели Файловая система, в поле Путь выбрать папку для импорта, нажав кнопку "...". Если сканер подключен по сети, папка должна соответствовать элементу в адресной книге при сканировании.
  6. Установить необходимое значение поля Разделение документов. Возможные значения поля Разделение документов:
    • Отключено - значение по умолчанию. Предполагает отключение функции разделения документов;
    • По штрих-коду - необходимо задать формат штрих-кода и значение;
    • По пустой странице - необходимо задать количество черных точек в процентном содержании; 
    • По числу страниц с указанием количества страниц в документе.
  7. Нажать OК.
  8. В рабочей области автоматически будут отображены файлы, расположенные в выбранной папке.
  9. Нажать кнопку Отправить. Документы будут отправлены на распознавание (стадия OCR). Документы обрабатываются с помощью описанных шаблонов, кроме того извлекаются значения для требуемых полей. После распознавания документа он автоматически перемещается На экспорт.
  10. Перейти на стадию Экспорт.
  11. Нажать кнопку Экспортировать. Окно системы FILLIN закрылось. Распознанные документы отобразились на данной форме.
  12. Отметить галочкой поле Экспортировать, если документ необходимо верифицировать и записать в базу данных. Кнопка Не экспортировать ничего снимает галочки со всех документов на вкладке, кнопка Экспортировать всё ставит галочки у всех документов на данной вкладке.

Верификация:

Чтобы документы после экспорта попали на верификацию, необходимо:

  1. На закладке Сканирование у каждого документа, который необходимо отправить на верификацию, поставить галочку в поле Экспортировать.
    • Кнопка Не экспортировать ничего - снимает выделение со всех документов; 
    • Кнопка Экспортировать все - ставит галочки в поле Экспортировать у каждого документа.
  2. Нажать кнопку Верифицировать. Откроется форма верификации.

Просмотр истории:

Записать историю в файл

Для сохранения истории в файл необходимо:

  1. Нажать кнопку История;
  2. В меню выбрать пункт Сохранить файл в историю;
  3. Откроется окнов для выбора папки, в которую будет записан файл с историей в формате .xml.

Прочитать историю из файла

В системе есть возможность загрузки истории из файла. Для этого необходимо:

  1. Нажать кнопку История;
  2. В меню выбрать пункт Прочитать историю из файла;
  3. Откроется окно для выбора папки, в которой расположен файл с историей в формате .xml;
  4. На вкладке История отобразятся документы, загруженные из файла.

Очистить историю

Чтобы очистить историю, необходимо:

  1. Перейти на вкладку История.
  2. Щелкнуть правой кнопкой мыши по документу.
  3. Появится контекстное меню. Выбрать пункт Удалить из истории.
  4. Документ будет удален из истории.

Экспорт документов в Alfresco

Для просмора сканированной копии документа необходимо выполнить действия:

  1. Перейти на вкладку История окна "FILLIN: Загрузка документов".
  2. Щелкнуть правой кнопкой мыши по нужному документу .
  3. В открывшемся контекстном меню выбрать пункт Открыть документ в Alfresco.
  4. Откроется окно браузера, выбранного по умолчанию, где отобразится отсканированная копия выбранного документа в системе Alfresco.
