Обработка "FILLIN:Загрузка документов"

Программное решение FILLIN:Загрузка документов может распознавать отсканированные документы и автоматически вносить полученные данные в базу 1С.

Основная форма обработки при запуске обработки из конфигурации Бухгалтерия предприятия 3.0. На форме находятся 5 основных страниц: "Значения по умолчанию", "Сканирование", "Верификация", "Словарь соответствий" и "История".

  • Значения по умолчанию - на данной вкладке устанавливаются значения по умолчанию, которые будут подставлены в документ на стадии верификации в случае, если соответствующие поля не будут распознаны.
  • Сканирование - на данной странице представлены все отсканированные документы.
  • Верификация - страница отображается при отправке документа из таблицы Сканирование на стадию верификации.
  • Словарь соответствий - на странице отображены все занесенные в базу данных соответствия значений справочников со строками для будущей подстановки при распознавании.
  • История - на странице отображены все документы, занесенные при помощи данной обработки, отсортированные по дате. При двойном клике по документу открывается соответствующий документ в базе данных 1С.

Значения по умолчанию:

На данной вкладке устанавливаются значения по умолчанию.

Вкладка, с которой будут подставлены значения (Покупка, Продажа) зависит от поля Вид документа.

Алгоритм определения Вида документа:

  • Если распознанное значение ИНН/КПП Заказчика из документа полностью совпадает с ИНН/КПП в справочнике Организации - вид документа Поступление.
  • Если распознанное значение ИНН/КПП Поставщика из документа полностью совпадает с ИНН/КПП в справочнике Организации - вид документа Реализация.
  • Если распознанное значение ИНН/КПП Поставщика из документа не найдено в справочнике Организации, но полностью совпадает с ИНН/КПП в справочнике Контрагенты - вид документа Реализация.
  • Если распознанное значение ИНН/КПП Заказчика из документа не найдено в справочнике Организации, но полностью совпадает с ИНН/КПП в справочнике Контрагенты - вид документа Поступление.

Если стоит галочка в поле Установливать в документе текущую дату, то при создании документа дата будет заменена на текущую.

Выбранное значение в поле Организация и Контрагент будет подставлено в документ, когда он будет экспортирован в 1С на вкладку Сканирование.

Установленное значение Организация/Контрагент не будет браться из Значений по умолчанию, если распознанное значение нашелось в базе в соответствующем справочнике.

Если для поля Организация нет нужного значения в справочнике, его можно добавить, нажав на кнопку Добавить организацию.

Если для поля Контрагент нет нужного значения в справочнике, его можно добавить, нажав на кнопку Добавить контрагента.

Сканирование:

Если сканер подключен локально, необходимо выполнить следующие действия:

  1. В окне "FILLIN: Загрузка документов" нажать кнопку Сканировать.
  2. Откроется приложение FILLIN COM SDK.
  3. Нажать Файл-->Настройки. Откроется окно настроек системы.
  4. Перейти на вкладку Импорт. На данной вкладке можно при необходимости установить флажки Автоматически отправлять когда документов с указанием количества документов (для автоматической отправки документов со стадии Импорт на последующие стадии) и Объединить в один документ при отправке (для объединения нескольких документов в один многостраничный документ).
  5. Выбрать тип цели Сканер, в поле Устройство указать необходимое устройство.
  6. Установить необходимое значение поля Разделение документов. Возможные значения поля Разделение документов:
    • Отключено - значение по умолчанию. Предполагает отключение функции разделения документов;
    • По штрих-коду - необходимо задать формат штрих-кода и значение;
    • По пустой странице - необходимо задать количество черных точек в процентном содержании;
    • По числу страниц с указанием количества страниц в документе.
  7. Нажать ОК.
  8. Отсканировать документы. Документы появятся в области отображения документов на стадии Импорт.
  9. Нажать кнопку Отправить. Документы будут отправлены на распознавание (стадия OCR). Документы обрабатываются с помощью описанных шаблонов, кроме того извлекаются значения для требуемых полей. После распознавания документа он автоматически перемещается На экспорт.
  10. Перейти на стадию Экспорт.
  11. Нажать кнопку Экспортировать. Окно системы FILLIN закрылось. Распознанные документы отобразились на данной форме на закладке Сканирование.
  12. Отметить галочкой поле Экспортировать, если документ необходимо верифицировать и записать в базу данных. Кнопка Не экспортировать ничего снимает галочки со всех документов на вкладке, кнопка Экспортировать всё ставит галочки у всех документов на данной вкладке.

Для того чтобы загрузить отсканированные документы из папки на локальном диске, необходимо выполнить следующие действия:

  1. В окне "FILLIN: Загрузка документов" нажать кнопку Сканировать.
  2. Откроется приложение FILLIN COM SDK.
  3. Открыть окно настроек системы.
  4. Перейти на вкладку Импорт. На данной вкладке при необходимости установить флажки Автоматически отправлять когда документов с указанием количества документов (для автоматической отправки документов со стадии Импорт на последующие стадии) и Объединить в один документ при отправке (для объединения нескольких документов в один многостраничный документ).
  5. Выбрать тип цели Файловая система, в поле Путь выбрать папку для импорта, нажав кнопку "...". Если сканер подключен по сети, папка должна соответствовать элементу в адресной книге при сканировании.
  6. Установить необходимое значение поля Разделение документов. Возможные значения поля Разделение документов:
    • Отключено - значение по умолчанию. Предполагает отключение функции разделения документов;
    • По штрих-коду - необходимо задать формат штрих-кода и значение;
    • По пустой странице - необходимо задать количество черных точек в процентном содержании; 
    • По числу страниц с указанием количества страниц в документе.
  7. Нажать OК.
  8. В рабочей области автоматически будут отображены файлы, расположенные в выбранной папке.
  9. Нажать кнопку Отправить. Документы будут отправлены на распознавание (стадия OCR). Документы обрабатываются с помощью описанных шаблонов, кроме того извлекаются значения для требуемых полей. После распознавания документа он автоматически перемещается На экспорт.
  10. Перейти на стадию Экспорт.
  11. Нажать кнопку Экспортировать. Окно системы FILLIN закрылось. Распознанные документы отобразились на данной форме.
  12. Отметить галочкой поле Экспортировать, если документ необходимо верифицировать и записать в базу данных. Кнопка Не экспортировать ничего снимает галочки со всех документов на вкладке, кнопка Экспортировать всё ставит галочки у всех документов на данной вкладке.

Верификация:

Эта страница отображается по нажатию на кнопку Верифицировать на основной форме обработки.

Тут отображаются атрибуты первого документа с закладки Сканирование. Форма верификации состоит из двух областей: реквизиты и табличная часть.

Реквизиты содержат исходные (взятые из результатов распознавания документов) и итоговые значения из базы. Исходные значения являются нередактируемыми, итоговые могут быть отредактированы пользователем. В табличной части документа строки с исходными значениями выделены бежевым цветом.

Поля могут быть помечены разными цветами:

  • красный - значение не найдено в базе;
  • зеленый - значение отличается от исходного;
  • розовый - необходимо сверить суммы;
  • оранжевый - единица измерения не сопадает с исходной.

На окне верификации располагаются следующие кнопки:

  • Добавить организацию (Создание организации) - добавление в справочник организаций новой организации; 
  • Добавить контрагента (Создание контрагента) - добавление в справочник контрагентов нового контрагента;
  • Открыть словарь соответствий (Словарь соответствий) - позволяет редактировать словарь соответствий;
  • Обновить полнотекстовый индекс - используется после добавления в справочник нового элемента (например, контрагента);
  • Работа с контактными данными - позволяет заменить адрес организации или контрагента в базе;
  • Выполнить (Запись и проведение документа) - в системе 1С будет создан документ на поступление товаров и услуг. Для того чтобы провести документ, необходимо нажать кнопку Ок;
  • Удалить документ - документ будет удален с закладки Сканирование. В окне верификации будет открыт следующий по порядку документ с закладки Сканирование, у которого стоит галочка в поле Экспортировать; 
  • Пропустить документ - на закладке Сканирование значение поля Экспортировать изменится на Нет (галочка будет снята). В окне верификации будет открыт следующий по порядку документ с закладки Сканирование, у которого стоит галочка в поле Экспортировать;
  • Закрыть - окно верификации закроется.

Проверка правильности распознавания

Подбор похожих значений

Подбор похожих значений осуществляется следующим образом:

  1. На этапе верификации выделить ячейку, помеченную красным цветом, и дважды кликнуть по ячейке. Появляются функциональные кнопки:
    • Выбрать из списка - открывает список наиболее похожих значений;
    • Открыть - открывает карточку объекта (организации, контрагента, товара, услуги);
    • Очистить - удаляет все данные, занесенные в ячейку (кнопка не доступна для табличной части документа).
  2. Нажать на кнопку Выбрать из списка. Появится список наиболее похожих значений из существующих в базе. Слева отображается процент схожести.
  3. Выбрать необходимое значение. Оно заменит исходное значение в ячейке, ячейка будет выделена зеленым.
  4. При выборе команды Выбрать значение из справочника откроется справочник Номенклатура.
  5. Выбрать необходимое значение из справочника. Ячейка пометится зеленым цветом.
  6. При нажатии правой кнопкой мыши по ячейке появляется контекстное меню, включающее сдедующие кнопки:
    • Подобрать рекомендуемое значение - открывает список подходящей номенклатуры, из которого можно выбрать необходимое значение;
    • Вернуть исходное значение - возвращает значение, которое было распознано изначально;
    • Создать номенклатуру (Создание номенклатуры) - позволяет создать новое значение в справочник Номенклатура (кнопка доступна только для табличной части документа).

Интеллектуальный подбор данных

Если на этапе верификации для какой-либо ячейки не было найдено значение в базе (ячейка выделена красным цветом), можно воспользоваться интеллектуальным подбором данных.

Для этого необходимо:

  1. Выбрать необходимую ячейку, выделенную красным цветом, и дважды щелкнуть левой кнопкой мыши. Появится возможность редактирования ячейки.
  2. Начать набирать значение в выделенной ячейке.
  3. Если значение найдено в базе, оно автоматически подставится в ячейку.
  4. Если значение не было найдено в базе, его можно добавить, нажав кнопку:
    • Добавить организацию (Создание организации) - для добавления организации в соответствующий справочник. После этого необходимо нажать кнопку Обновить полнотекстовый индекс;
    • Добавить контрагента (Создание контрагента) - для добавления контрагента в справочник Контрагентов. После этого необходимо нажать кнопку Обновить полнотекстовый индекс;
    • Создать номенклатуру (Создание номенклатуры) - для добавления наименования товара или услуги в справочник Номенклатура. Созданное наименование появится в табличной части документа в соответствующей ячейке.

Добавление новой записи в табличную часть

Чтобы создать новую запись в табличной части, необходимо выполнить следующие действия:

  1. В меню над табличной частью нажать кнопку Добавить. В конце таблицы добавятся две новые строки. Одна строка - исходное значение, другая строка - итоговое значение. Строка исходных значений является нередактируемой.
  2. Дважды щелкнуть по каждой ячейке новой строки, заполнить их значениями, введенными с клавиатуры или выбранными из справочника. Строка будет выделена зеленым цветом.

Создание номенклатуры

Номенклатуру можно создать вручную или автоматически.

  1. Чтобы создать номенклатуру вручную, необходимо выполнить следующие действия:
    1. В табличной части выделить ячейку колонки Номенклатура, помеченную красным цветом
    2. В выбранной ячейке ввести название нового товара или услуги. Нажать кнопку Создать номенклатуру (справа над табличной частью документа) или щелкнуть правой кнопкой мыши по ячейке и вызвать контекстное меню, в котором выбрать пункт Создать номенклатуру. Откроется окно создания новой номенклатуры.
    3. Заполнить открывшееся окно создания новой номенклатуры
    4. Нажать кнопку OK. В справочнике номенклатуры будет создана новая номенклатура. В табличной части выделенная ранее ячейка помечена зеленым цветом.
  2. Чтобы создать номенклатуру автоматически, необходимо выполнить следующие действия:
    1. На стадии верификации нажать кнопку Выполнить.
    2. В появившемся диалоговом окне нажать Да. В системе 1С будет создан документ. Все строки табличной части, наименования которых были помечены красным, будут созданы в справочнике номенклатуры.

Пересчет сумм

Пересчет сумм осуществляется следующим образом:

  1. В окне верификации отметить поле Всегда автоматически пересчитывать суммы в табличной части, в противном случае, при каждом пересчете будет всплывать диалоговое окно с потверждением пересчета.
  2. Изменить значение ячейки в колонке Количество/Цена/Сумма/Ставка НДС.
  3. Нажать клавишу Enter. Значения всех последующих ячеек строки пересчитаются в соответствии с внесенными изменениями. Все измененные вычисляемые ячейки пометятся зеленым цветом.
  4. Для ячеек, выделенных красным цветом, можно вызвать правой кнопкой мыши контекстное меню и выбрать пункт Подобрать рекомендуемое значение. Далее система вычислит необходимое значение и предложит его подставить. Измененная ячейка будет помечена зеленым цветом, а значения всех последующих ячеек автоматически пересчитаются.
  5. В модуле есть возможность сравнить исходное и расчетное значение суммы, как с учетом НДС, так и без учета НДС. Исходное значение основывается на исходных, распознанных значениях. Величина расчетного значения зависит от внесенных изменений.

Словарь соответствий:

При обработке табличной части документа на этапе верификации система ищет в номенклатуре точное совпадение с учетом зависимых полей. В случае если значение не найдено, оно ищется в словаре соответствий.

В данном окне доступны следующие кнопки:

  • Открыть файл словаря соответствий - откроется окно для выбора файла словаря соответствий;
  • Сохранить файл словаря соответствий - откроется окно для выбора папки, куда будет сохранен файл словаря соответствий в формате .xml.

В словаре соответствий имеется три вкладки:

  •  Номенклатура - содержит перечень наименований номенклатуры. Каждое значение в столбце Строка (где содержатся исходные распознанные значения) при распознавании будет заменено на соответствующее значение из столбца Номенклатура (из справочника Номенклатура);
  • Организация - содержит перечень наименований организаций. Каждое значение в столбце Строка (где содержатся исходные распознанные значения) при распознавании будет заменено на соответствующее значение из столбца Организация (из справочника 1С);
  • Контрагенты - содержит перечень наименований контрагентов. Каждое значение в столбце Строка (где содержатся исходные распознанные значения) при распознавании будет заменено на соответствующее значение из столбца Контрагент (из справочника Контрагенты).

Для справочника соответствий доступно контекстное меню, вызываемое нажатие правой кнопкой мыши по строке. Основные функции:

  • Добавить - добавляет новую строку в справочник соответствий;
  • Скопировать - копирует выбранную строку из справочника соответствий;
  • Изменить - позволяет отредактировать выбранную запись;
  • Удалить - удаляет выбранную запись из справочника соответствий;
  • Переместить вверх - перемещает запись на одну строку вверх;
  • Переместить вниз - перемещает запись на одну строку вниз;
  • Также доступны функции сортировки и отбора

Если заменить любое распознанное значение на значение из справочника, эта замена автоматически добавится в словарь соответствий. Впоследствии система будет автоматически заменять эти значения, беря соответствие из справочника.

Просмотр истории:

Записать историю в файл

Для сохранения истории в файл необходимо:

  1. Нажать кнопку История;
  2. В меню выбрать пункт Сохранить файл в историю;
  3. Откроется окнов для выбора папки, в которую будет записан файл с историей в формате .xml.

Прочитать историю из файла

В системе есть возможность загрузки истории из файла. Для этого необходимо:

  1. Нажать кнопку История;
  2. В меню выбрать пункт Прочитать историю из файла;
  3. Откроется окно для выбора папки, в которой расположен файл с историей в формате .xml;
  4. На вкладке История отобразятся документы, загруженные из файла.

Очистить историю

Чтобы очистить историю, необходимо:

  1. Перейти на вкладку История.
  2. Щелкнуть правой кнопкой мыши по документу.
  3. Появится контекстное меню. Выбрать пункт Удалить из истории.
  4. Документ будет удален из истории.

Экспорт документов в Alfresco

Для просмора сканированной копии документа необходимо выполнить действия:

  1. Перейти на вкладку История окна "FILLIN: Загрузка документов".
  2. Щелкнуть правой кнопкой мыши по нужному документу .
  3. В открывшемся контекстном меню выбрать пункт Открыть документ в Alfresco.
  4. Откроется окно браузера, выбранного по умолчанию, где отобразится отсканированная копия выбранного документа в системе Alfresco.
